# 🌲 SABD2026 — Machine Learning Project

## 📌 Описание проекта

Данный проект выполнен в рамках курса SABD2026 и посвящён исследованию возможностей применения различных алгоритмов машинного обучения для решения задачи многомерной регрессии. Основная цель работы — построение и сравнительный анализ моделей, способных предсказывать энергетические спектры на основе нормализованных входных параметров.

В рамках проекта были реализованы, обучены и протестированы три различные модели машинного обучения: Random Forest Regressor, Extra Trees Regressor и Hist Gradient Boosting Regressor. Для каждой модели проведён детальный анализ метрик качества, изучена важность признаков и выполнено сравнение результатов.

## 🎯 Постановка задачи и обоснование выбора моделей

### Формулировка задачи

Необходимо на основе 10 входных признаков предсказать 60 целевых переменных, представляющих собой значения энергетического спектра ISO Cf-252. Данная задача относится к классу задач многомерной регрессии (Multi-output Regression), где требуется одновременно предсказать несколько непрерывных целевых переменных.

### Обоснование выбора алгоритмов

При выборе моделей для исследования я руководствовалась несколькими критериями:

1. **Разнообразие подходов** — хотелось сравнить различные семейства алгоритмов (ансамблевые методы на основе деревьев решений)
2. **Интерпретируемость** — возможность анализа важности признаков и понимания того, как модель принимает решения
3. **Устойчивость к переобучению** — способность моделей работать с достаточно большим датасетом (500+ тысяч записей)
4. **Вычислительная эффективность** — возможность обучения за разумное время на стандартном оборудовании

#### Random Forest (Случайный лес)

Random Forest был выбран в качестве базовой модели по следующим причинам:

- **Хорошая отправная точка** — этот алгоритм зарекомендовал себя как надёжный и эффективный метод для широкого круга задач регрессии
- **Устойчивость к шуму** — благодаря механизму бэггинга (bagging) и случайному выбору признаков, модель демонстрирует высокую устойчивость к переобучению
- **Отсутствие необходимости масштабирования** — деревья решений не требуют предварительной нормализации данных, что упрощает предобработку
- **Интерпретируемость** — возможность получить оценку важности каждого признака, что критически важно для понимания природы данных
- **Параллелизация** — алгоритм эффективно использует все доступные ядра процессора, что ускоряет процесс обучения

#### Extra Trees (Extremely Randomized Trees)

Extra Trees представляет собой модификацию Random Forest с повышенной степенью рандомизации. Выбор этой модели обусловлен желанием исследовать:

- **Влияние случайности** — Extra Trees использует случайные разбиения вместо оптимальных, что теоретически должно снизить дисперсию модели и ускорить обучение
- **Сравнение с базовой моделью** — интересно было понять, даст ли дополнительная рандомизация выигрыш в качестве или, наоборот, ухудшит результаты
- **Скорость работы** — алгоритм должен обучаться быстрее классического Random Forest за счёт отсутствия поиска оптимальных разбиений

#### Hist Gradient Boosting (Гистограммный градиентный бустинг)

Hist Gradient Boosting был включён в исследование как представитель семейства бустинговых алгоритмов:

- **Потенциально более высокое качество** — градиентный бустинг часто показывает лучшие результаты по сравнению с бэггинговыми методами
- **Эффективность на больших данных** — использование гистограммного подхода (группировка значений признаков в бины) существенно ускоряет обучение на больших датасетах
- **Последовательное обучение** — в отличие от Random Forest, где деревья строятся независимо, бустинг строит деревья последовательно, каждое следующее исправляет ошибки предыдущего, что может дать выигрыш в точности
- **Современный подход** — это относительно новый и перспективный алгоритм, который заслуживает изучения

## 📊 Описание датасета

### Общие характеристики

- **Источник данных**: файл `normalized_inputs_and_targets_gsf.csv`
- **Общий объём**: 500 713 записей (строк)
- **Количество признаков**: 70 столбцов, из которых:
  - 10 входных признаков (features)
  - 60 целевых переменных (targets)

### Входные признаки (X)

Входные данные представляют собой нормализованные параметры, обозначенные как `Xin_norm`:

- `0in_norm`, `2in_norm`, `3in_norm`, `5in_norm`, `6in_norm`
- `8in_norm`, `10in_norm`, `12in_norm`, `15in_norm`, `18in_norm`

Эти признаки, вероятно, соответствуют различным параметрам измерительной установки или результатам предварительных измерений в определённых точках. Нормализация данных выполнена заранее, что позволяет моделям работать с признаками в сопоставимых масштабах.

### Целевые переменные (y)

Целевые переменные представляют собой нормализованные значения интенсивности энергетического спектра в 60 энергетических каналах:

- `E1_norm`, `E2_norm`, ..., `E60_norm`

Энергетическая шкала охватывает диапазон от приблизительно 10⁻⁹ МэВ до 631 МэВ, что соответствует стандартному спектру ISO Cf-252 (калифорний-252 — источник нейтронов). Значения распределены по логарифмической шкале, что характерно для спектроскопических данных.

### Разделение данных

Для обучения и тестирования моделей данные были разделены следующим образом:

- **Обучающая выборка**: 350 499 образцов (70% от общего объёма)
- **Тестовая выборка**: 150 214 образцов (30% от общего объёма)

Разделение выполнено случайным образом с фиксированным параметром `random_state=42` для обеспечения воспроизводимости результатов.

## ⚙️ Детальное описание моделей

### Random Forest Regressor

#### Архитектура модели

Random Forest — это ансамблевый метод, который строит множество деревьев решений и усредняет их предсказания. Основные принципы работы:

- **Бэггинг (Bootstrap Aggregating)** — каждое дерево обучается на случайной подвыборке данных (с возвращением)
- **Случайный выбор признаков** — при каждом разбиении узла рассматривается только случайное подмножество признаков
- **Голосование** — итоговое предсказание формируется как среднее арифметическое предсказаний всех деревьев

#### Гиперпараметры модели

```python
RandomForestRegressor(
    n_estimators=50,           # Количество деревьев в лесу
    max_depth=15,              # Максимальная глубина каждого дерева
    min_samples_split=10,      # Минимальное число образцов для разделения узла
    min_samples_leaf=5,        # Минимальное число образцов в листе
    random_state=42,           # Фиксация случайности
    n_jobs=-1                  # Использование всех ядер процессора
)
```

**Обоснование выбора гиперпараметров:**

- `n_estimators=50` — компромисс между качеством предсказаний и временем обучения. Большее количество деревьев могло бы улучшить результат, но увеличило бы время обучения
- `max_depth=15` — ограничение глубины деревьев предотвращает переобучение. Без ограничения деревья могли бы запомнить обучающую выборку, но плохо обобщать на новых данных
- `min_samples_split=10` и `min_samples_leaf=5` — дополнительные механизмы регуляризации, которые не позволяют деревьям становиться слишком сложными
- `random_state=42` — обеспечивает воспроизводимость результатов при повторных запусках

### Extra Trees Regressor

#### Отличия от Random Forest

Extra Trees (Extremely Randomized Trees) вносит дополнительную случайность в процесс построения деревьев:

- **Случайные разбиения** — вместо поиска оптимальной точки разделения для каждого признака, Extra Trees выбирает случайные точки разделения
- **Весь датасет** — в отличие от Random Forest, Extra Trees может использовать весь датасет для обучения каждого дерева (без бэггинга), хотя это настраиваемый параметр

#### Гиперпараметры модели

```python
ExtraTreesRegressor(
    n_estimators=50,           # Количество деревьев в лесу
    max_depth=15,              # Максимальная глубина каждого дерева
    min_samples_split=10,      # Минимальное число образцов для разделения узла
    min_samples_leaf=5,        # Минимальное число образцов в листе
    random_state=42,           # Фиксация случайности
    n_jobs=-1                  # Использование всех ядер процессора
)
```

Параметры выбраны идентичными Random Forest для обеспечения честного сравнения моделей.

### Hist Gradient Boosting Regressor

#### Особенности реализации

Hist Gradient Boosting — это оптимизированная версия градиентного бустинга, которая использует гистограммный подход:

- **Гистограммы** — непрерывные признаки дискретизируются (группируются в бины), что существенно ускоряет поиск лучших разбиений
- **Последовательное обучение** — каждое новое дерево строится для исправления ошибок предыдущих деревьев
- **MultiOutputRegressor** — поскольку HistGradientBoostingRegressor не поддерживает многомерный вывод напрямую, используется обёртка MultiOutputRegressor, которая обучает отдельную модель для каждой из 60 целевых переменных

#### Гиперпараметры модели

```python
HistGradientBoostingRegressor(
    max_iter=100,              # Количество деревьев в ансамбле
    max_depth=10,              # Максимальная глубина деревьев
    learning_rate=0.1,         # Скорость обучения
    l2_regularization=0.1,     # L2-регуляризация
    max_bins=255,              # Количество бинов для дискретизации
    random_state=42            # Фиксация случайности
)
```

**Обоснование выбора гиперпараметров:**

- `max_iter=100` — количество деревьев в ансамбле. В бустинге обычно требуется меньше деревьев, чем в Random Forest
- `max_depth=10` — деревья в бустинге обычно делают мельче, чем в Random Forest, так как каждое дерево исправляет ошибки предыдущих
- `learning_rate=0.1` — скорость обучения определяет, насколько сильно каждое новое дерево влияет на итоговое предсказание. Меньшее значение требует больше итераций, но может дать лучший результат
- `l2_regularization=0.1` — штраф за большие веса, помогает бороться с переобучением
- `max_bins=255` — максимальное количество бинов для дискретизации признаков

### Интерпретация метрик

#### R² (Коэффициент детерминации)

R² показывает, насколько хорошо модель объясняет дисперсию данных:

- **Hist Gradient Boosting (0.7843)** — модель объясняет приблизительно 78.43% дисперсии целевых переменных. Это **наилучший результат** среди всех протестированных моделей
- **Random Forest (0.7770)** — объясняет 77.70% дисперсии, отставание от лидера составляет всего 0.73 процентных пункта, что является незначительной разницей
- **Extra Trees (0.7646)** — объясняет 76.46% дисперсии, показывает несколько худший результат по сравнению с топ-моделями

#### MSE (Mean Squared Error — Среднеквадратичная ошибка)

MSE измеряет средний квадрат разности между предсказанными и фактическими значениями:

- **Random Forest (0.0018)** — наименьшая ошибка среди всех моделей
- **Extra Trees (0.0019)** — незначительно уступает Random Forest
- **Hist Gradient Boosting (0.0020)** — показывает несколько большую ошибку, но разница минимальна

#### MAE (Mean Absolute Error — Средняя абсолютная ошибка)

MAE измеряет среднюю абсолютную разность между предсказаниями и фактическими значениями:

- **Random Forest (0.0145)** — в среднем ошибка предсказания составляет 1.45% от диапазона значений. Наилучший результат
- **Hist Gradient Boosting (0.0147)** — очень близкий результат к Random Forest (отличие всего 1.4%)
- **Extra Trees (0.0151)** — несколько большая ошибка

#### Стандартное отклонение R² (Std R²)

Показывает стабильность модели по всем 60 целевым переменным:

- **Hist Gradient Boosting (0.0965)** — наименьшее стандартное отклонение указывает на то, что модель демонстрирует наиболее стабильное качество предсказаний для всех целевых переменных
- **Random Forest (0.1196)** — приемлемая стабильность
- **Extra Trees (0.1274)** — наибольшая вариативность качества по разным целевым переменным

- Приемлемое время обучения (~33 сек)

**✅ Random Forest показывает отличные результаты и является надёжной альтернативой**

- Второе место по R² (0.7770), отставание минимально
- Наилучшие результаты по MSE и MAE
- Проще в интерпретации
- Самая медленная модель (~64 сек)

**✅ Extra Trees — самый быстрый, но с небольшим отставанием по качеству**

- Третье место по R² (0.7646)
- Самое быстрое обучение (~26 сек)
- Незначительное отставание по качеству от топ-моделей

### Рекомендации по выбору модели

Для данной задачи **рекомендуется использовать Hist Gradient Boosting** в качестве основной модели по следующим причинам:

1. **Лучшее качество предсказаний** — модель объясняет наибольшую долю дисперсии данных
2. **Стабильность** — демонстрирует наиболее стабильные результаты по всем целевым переменным
3. **Превосходство на сложных случаях** — значительно лучше справляется с "трудными" целевыми переменными, что может быть критически важно для практического применения
4. **Приемлемая скорость** — несмотря на большее время обучения, модель остаётся практичной

**Random Forest** рекомендуется в качестве альтернативы, если:

- Важна простота интерпретации модели
- Требуется минимальная ошибка (MSE/MAE)
- Время обучения не критично

**Extra Trees** рекомендуется, если:

- Нужно максимально быстрое обучение
- Важно быстрое прототипирование
- Допустимо небольшое снижение качества
